NAME

  DupCookieAckInShutSend.seq - Duplicate COOKIE-ACK chunk is received in SHUTDOWN-SENT state

PURPOSE

  To check that if COOKIE-ACK chunk is received in SHUTDOWN-SENT state
  then the endpoint should silently discard a received COOKIE-ACK chunk.

SYNOPSIS

  ./DupCookieAckInShutSend.seq [-tooloption ...] -pkt ./DupCookieAckInShutSend.def
    -tooloption : v6eval tool option
  See Also: ../common/STD_PKT_COMMON.def
            ../common/SCTP_COMMON.def

PRE-TEST CONDITION

  Association is not established between endpoint A and B. Arrange the
  data in endpoint A such that COOKIE-ACK message is sent to endpoint
  B after receiving SHUTDOWN message.

TEST PROCEDURE

  Endpoint A                           Endpoint B                ULP
  (ESTABLISHED)                        (ESTABLISHED)
                <----------------       SHUTDOWN
  COOKIE-ACK    ----------------->      Silently discard
                <----------------       SHUTDOWN
  TEST DESCRIPTION:
  1. Send COOKIE-ACK message after received SHUTDOWN message.
  2. Check A:COOKIE-ACK message received in SHUTDOWN-SENT state
     is discarded.

NOTE

  None

REFERENCE

  RFC 4960
  5.2.5.  Handle Duplicate COOKIE-ACK.
    At any state other than COOKIE-ECHOED, an endpoint should silently
    discard a received COOKIE ACK chunk.
